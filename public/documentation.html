The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

What this api does

Business Logic URL HTTP Method Request body data format Response body data format
Return a list of ALL movies to the user /movies GET none A JSON object holding all the movies and data
Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user /movies/:Title GET none A JSON object holding all the data of the requested movie
Return data about a genre (description) by name/title (e.g., “Thriller”) /movies/:Genre GET none A JSON object holding the data about a genre
Return data about a director (bio, birth year, death year) by name /movies/Directors/:Name GET none A JSON object holding all the information about the requested director
Allow new users to register /Users POST A JSON Object holding data about the user to add.
{ name: 'Harry Potter', password: 'Quidditsch' } 		A JSON Object holding data about the added user including an ID
Allow users to update their user info (username) /Users/:Username PUT None A text message indicating the user info was successfully updated
Allow users to add a movie to their list of favorites /Watchlist/:_id POST A JSON object holding the following information about the added movie: title, director(name, birth, death), genres.
Looking like this: { title: 'Harry Potter 7', director: { name: 'J.K. Rowling (not really)', birth:'Long long ago', death:'quite a while ago, I think' }, genre: 'fantasy' } 		a message indicating the movie was successfully added to the watchlist
Allow users to remove a movie from their list of favorites Users/Watchlist/:_id DELETE None a message indicating the movie was successfully deleted from the watchlist
Allow existing users to deregister /Users/:Username DELETE None A goodbye message
